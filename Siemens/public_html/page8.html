Лекция 8: отладка и тестирование STL-программ

 

Введение в отладку и тестирование STL-программ

Почему важна отладка

Отладка – ключевая стадия разработки программ для ПЛК. Даже небольшая ошибка в логике может привести к нештатной работе оборудования. Отладка позволяет:

Найти ошибки на ранних этапах.

Проверить корректность алгоритмов.

Убедиться в надежности системы при реальном или смоделированном использовании.

Особенности отладки STL-кода

Язык STL – низкоуровневый, командно-ориентированный язык, где каждая строка кода выполняется последовательно. Ошибки могут быть неочевидными из-за отсутствия явных конструкций условий и циклов, а использование логического стека делает отладку более трудоемкой. Для эффективного тестирования используются:

Симуляторы, такие как PLCSIM.

Таблицы наблюдения (Watch Table) для отслеживания значений входов/выходов.

Мониторинг состояния внутренних регистров и битовых переменных.

Инструменты отладки в Step 7

Симулятор PLCSIM

PLCSIM позволяет моделировать работу ПЛК без физического оборудования:

Запуск программы в режиме симуляции: даёт возможность анализировать переходы и изменение значений в реальном времени.

Тестирование на виртуальном контроллере: выявление логических ошибок без риска для оборудования.

Таблицы наблюдения (Watch Tables)

Мониторинг переменных: отслеживание значений входов, выходов, внутренних регистров и аккумулятора.

Логирование изменений: запись последовательности изменений для последующего анализа.

Режим пошагового выполнения

Пошаговый (step-by-step) режим: позволяет выполнять программу по одной строке, анализируя изменения состояния после каждой команды.

Breakpoints (точки останова): установка точек остановки для проверки конкретных участков кода.

Симулятор PLCSIM

Использование встроенных средств диагностики

Системные сообщения и индикаторы: отображают ошибки и предупреждения на экране.

Средства профилирования: помогают анализировать время выполнения отдельных участков кода.

Методики отладки STL-программ

Комментарии и структурирование кода

Параллельное документирование: Каждая строка кода должна сопровождаться подробным комментарием, объясняющим логику (например, «A I0.0 – загрузка значения входа», «= Q0.0 – запись результата в выход»).

Использование меток: для переходов (JU, JC, JCN) удобно применять осмысленные метки, которые упростят навигацию по коду.

Разбиение программы на модули

Модульное тестирование: Тестируйте отдельные модули (FB, FC) перед интеграцией в основной блок OB.

Локальное тестирование: если возможно, создавайте тестовые проекты для проверки небольших фрагментов логики.

Применение симуляторов

Запуск программы в PLCSIM: необходимо загружать и запускать программу в симуляционном режиме, чтобы проверить реакции на изменение входных сигналов.

Эмуляция внешних воздействий: Изменяйте значения входов вручную или с помощью скриптов симулятора для имитации различных сценариев работы.

Логгирование и трассировка

Запись изменений: В процессе отладки записывайте последовательность изменений значений переменных и регистров.

Анализ ошибок: Используйте системные логи для выявления ошибок, таких как переход в неправильное состояние или некорректное выполнение арифметических операций.
