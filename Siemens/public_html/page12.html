Лабораторная работа 4: Обработка аварийных сигналов и приоритеты прерываний

 

Цель: изучить обработку аварийных событий через аварийные OB (например, OB82) и приоритетное выполнение программ.

Теоретическая часть

1.     Аварийные OB-блоки:

OB82 — обработка диагностических событий модулей (например, обрыв датчика).

OB80 — контроль времени цикла.

OB35 — циклические прерывания (например, каждые 100 мс).

2.     Приоритеты выполнения:

Аварийные OB имеют более высокий приоритет, чем OB1.

Прерывание OB1 при возникновении события.

 

Задание

1.     Настройка аварийного OB:

Создайте OB82 для обработки ошибки модуля ввода/вывода.

При срабатывании OB82 должен:

Записать код ошибки в DB1.

Активировать аварийный выход Q0.7.

Отправить сообщение на HMI (эмулировать через DB).

2.     Имитация аварии:

В OB1 добавьте искусственную ошибку (например, деление на ноль при нажатии кнопки I0.3).

Проверьте, что OB82 корректно обрабатывает прерывание.

3.     Приоритеты:

Добавьте OB35 с циклом 500 мс для мигания диагностическим светодиодом Q0.6.

Убедитесь, что OB35 не блокируется OB1.

Пример кода

OB1 (Искусственная ошибка):

NETWORK 1:

    A     I      0.3          // Кнопка "Генерация ошибки"

    JC    GENERATE_ERROR

    JU    END

 

GENERATE_ERROR:

    L     0                   // Деление на ноль

    L     0

    /I   

    T     MW100               // Здесь возникнет ошибка

 

END: NOP 0

OB82 (Обработка аварии):

NETWORK 1:

    L     OB82_EV_CLASS       // Класс события

    T     DB1.DBW0            // Сохранить в DB1

    S     Q      0.7          // Аварийный выход

    L     16#55AA             // Код диагностики

    T     DB1.DBW2

OB35 (Диагностический сигнал):

NETWORK 1:

    L     DB1.DBX10.0         // Флаг состояния

    INV                     

    T     DB1.DBX10.0

    =     Q      0.6          // Мигающий светодиод

Порядок выполнения

1.     Создайте проект в Simatic manager, добавьте OB1, OB35 и OB82.

2.     Настройте DB1 для хранения данных:

DB1.DBW0 — код ошибки.

DB1.DBW2 — дополнительный статус.

DB1.DBX10.0 — флаг для мигания.

3.     Запустите симуляцию:

Активируйте I0.3 для генерации ошибки — проверьте реакцию OB82.

Убедитесь, что OB35 продолжает работать независимо от ошибки.
